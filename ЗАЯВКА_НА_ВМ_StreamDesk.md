## Характеристики ВМ

1. **Hostname** - streamdesk-prod
2. **CPU** - 4 ядра
3. **RAM** - 8 GB
4. **HDD** - 150 GB
5. **OS** - Ubuntu 22.04 LTS
6. **Подсеть** - С доступом в интернет (публичный доступ по HTTPS)

* **Кто будет подключаться**: 
  - Подключения будут проводиться внутри сети УИ и извне - SSH (для администрирования)
  - Пользователи будут подключаться через веб-интерфейс по HTTPS

* **Если требуется внешний доступ, то какой доступ - ssh/http(s). Требуется ли опубликовать в общем доступе в интернет, например https, либо необходима доступность только внутри рабочей сети, либо в целом внутри университета?**
  
  Требуется публичный доступ из интернет по HTTPS (порт 443). Приложение должно быть доступно для пользователей как внутри сети УИ, так и из внешней сети интернет.

* **Если требуется публичный доступ из интернет http/https, то необходимо указать к каким именно системам (можно просто ip адресам) требуется доступ с этой ВМ, каким портам. Если только внутри из рабочих сетей, то не требуется**
  
  Требуется исходящий доступ с ВМ к следующим ресурсам:
  - **vMix API** (если используется внешний vMix сервер): порт 8088 (HTTP)
  - **Внешние API для поиска оборудования**: стандартные HTTPS порты (443) для веб-скрапинга и парсинга информации об оборудовании
  - **GitHub/GitLab** (для обновлений кода): порт 443 (HTTPS)
  - **npm registry** (для установки зависимостей): порт 443 (HTTPS)
  - **Let's Encrypt** (для SSL сертификатов): порт 80, 443 (HTTP/HTTPS)

* **Требуется ли доступ с этой (вм/приложения) на другие ресурсы УИ или ВМ/серверы:**
  
  Доступ к внутренним ресурсам УИ не требуется. Приложение является автономным и использует собственную базу данных PostgreSQL, развернутую на этой же ВМ.

* **Обоснование:**
  
  Для развертывания и эксплуатации CRM системы "StreamDesk" - комплексной системы управления стриминговой студией. Система предназначена для:
  
  - Управления календарем событий и бронированием оборудования
  - Учета и управления инвентарем оборудования (камеры, микрофоны, компьютеры и т.д.)
  - Мониторинга состояния систем и серверов
  - Управления стримами и интеграции с платформами
  - Создания схем подключения оборудования с визуализацией
  - Интеграции с vMix для управления видеопроизводством
  - Управления задачами и проектами
  - Системы уведомлений
  
  Система рассчитана на одновременную работу до 20 пользователей. Технологический стек: Node.js (Express.js), React, PostgreSQL, Nginx. Требуется стабильная работа без задержек при одновременном использовании.

## Примечания по ресурсам

**Обоснование выбранных характеристик:**

- **4 CPU ядра**: Node.js приложение с Express.js, PostgreSQL база данных и Nginx reverse proxy требуют достаточной вычислительной мощности для обработки запросов от 20 одновременных пользователей без задержек.

- **8 GB RAM**: 
  - Node.js процесс: ~2 GB
  - PostgreSQL: ~2 GB
  - Nginx: ~500 MB
  - Система Ubuntu: ~1 GB
  - Запас для пиковых нагрузок: ~2.5 GB

- **150 GB HDD**:
  - Ubuntu 22.04: ~20 GB
  - Приложение и зависимости: ~5 GB
  - PostgreSQL база данных с данными: ~20 GB
  - Логи приложения и системы: ~10 GB
  - Запас для роста данных и бэкапов: ~95 GB

- **Ubuntu 22.04 LTS**: Стабильная LTS версия с долгосрочной поддержкой, оптимальная для production окружения.
