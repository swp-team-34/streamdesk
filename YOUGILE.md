# Интеграция YouGile с StreamDesk

Таск-менеджер StreamDesk может работать с [YouGile](https://ru.yougile.com) через REST API v2. Локальные задачи при этом не затрагиваются: всё продолжает работать как раньше. YouGile подключается опционально.

## Документация API

- **YouGile REST API v2:** https://ru.yougile.com/api-v2#/

## Шаги для администратора

### 1. Получить ключ API YouGile

1. Зайти в YouGile под учётной записью компании.
2. По документации: получить ID компании (запрос к API с логином и паролем) и затем получить ключ API (логин, пароль, `companyId`).
3. Либо использовать раздел в личном кабинете YouGile для создания ключа API, если он там есть.
4. Ключ не имеет срока действия. Ограничение: **не более 50 запросов в минуту** на компанию.

### 2. Настроить StreamDesk

В корне проекта в файле `.env` (скопировать с `.env.example`, если его ещё нет) добавить:

```env
YOUGILE_API_KEY=ваш_ключ_yougile
```

Опционально (если нужен другой хост):

```env
YOUGILE_BASE_URL=https://yougile.com/api-v2
```

После сохранения `.env` перезапустить сервер (например, `npm run dev` или сервис на сервере).

### 3. Проверка

- В приложении: **Настройки → Интеграции** — блок «YouGile (таск-менеджер)». Должно быть «Подключён» и кнопка «Проверить подключение» покажет количество проектов.
- Либо запрос с сервера: `GET /api/yougile/status` → `{ "configured": true }`, затем `GET /api/yougile/projects` — список проектов.

## Эндпоинты StreamDesk (прокси к YouGile)

Все запросы идут на ваш сервер; ключ API хранится только в `.env` на сервере.

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/yougile/status` | Проверка: настроен ли YouGile |
| POST | `/api/yougile/auth/key` | Получить ключ (body: `login`, `password`, `companyId`) |
| POST | `/api/yougile/auth/companies` | Список компаний (body: `login`, `password`) |
| GET | `/api/yougile/projects` | Список проектов |
| GET | `/api/yougile/projects/:id` | Один проект |
| GET | `/api/yougile/boards?projectId=` | Доски (опционально по проекту) |
| GET | `/api/yougile/columns?boardId=` | Колонки (опционально по доске) |
| GET | `/api/yougile/tasks?projectId=&boardId=&columnId=` | Список задач |
| GET | `/api/yougile/tasks/:id` | Одна задача |
| POST | `/api/yougile/tasks` | Создать задачу (body по формату YouGile) |
| PUT | `/api/yougile/tasks/:id` | Обновить задачу |
| DELETE | `/api/yougile/tasks/:id` | Удалить задачу |

Точные форматы полей для задач и других сущностей см. в [документации YouGile](https://ru.yougile.com/api-v2#/).

## Поведение приложения

- **Без `YOUGILE_API_KEY`:** таск-менеджер работает только с локальными задачами (как раньше).
- **С ключом:** в настройках отображается статус YouGile, можно просматривать проекты/доски/колонки/задачи и создавать/обновлять/удалять задачи в YouGile через эти API. Локальные задачи и разделы (Мои задачи, По проектам и т.д.) не меняются.

При необходимости можно доработать интерфейс «Задачи»: переключение источника «Локально / YouGile», импорт из YouGile в локальные задачи или синхронизацию в обе стороны.
