# Деплой на сервер

**Можно просто заменить на сервере все файлы проекта.** Важно только не перезаписывать серверный `.env` (там свои DATABASE_URL и PORT).

---

## 1. Сборка на своём компьютере

В корне проекта выполните:

```bash
npm run build
```

Будет создана папка `dist` и `client/dist` с готовым приложением.

## 2. Что загружать в FileZilla

Подключитесь к серверу (SFTP), откройте на сервере каталог проекта (например `/opt/streamdesk`).

**Загрузите всё содержимое папки проекта**, кроме:

| Не загружать |
|--------------|
| `node_modules` |
| `client/node_modules` |
| `.git` |
| **`.env`** — на сервере оставьте свой .env (БД, PORT), не заменяйте его |
| `attached_assets` (если не нужны локальные вложения) |
| `design-website` |
| `*.log` |

То есть загружайте: `client/`, `server/`, `shared/`, `dist/`, `client/dist/`, `package.json`, `deploy-to-server.sh`, все конфиги и остальные файлы. Папки `node_modules` можно не тянуть — на сервере выполните `npm install --production`.

## 3. После загрузки — на сервере

По SSH зайдите на сервер и выполните:

```bash
cd /opt/streamdesk
chmod +x deploy-to-server.sh
./deploy-to-server.sh
```

Скрипт поставит зависимости (если нужно), проверит/соберёт клиент при необходимости и перезапустит приложение через PM2.

---

**Кратко:**  
1. Локально: `npm run build`  
2. FileZilla: залить всё в папку проекта на сервере, **не затирать** серверный `.env`  
3. SSH: `cd /opt/streamdesk` → `./deploy-to-server.sh`  
4. Проверить: `pm2 status` и открыть сайт в браузере
