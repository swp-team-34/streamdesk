# Настройка сервера StreamDesk

Инструкция для развёртывания на вашем сервере (PostgreSQL, регистрация пользователей, синхронизация с YouGile с учётом лимитов API).

---

## Автоматическая настройка (один раз)

С вашего компьютера можно один раз поднять всё на сервере и затем деплоить код.

1. **Создайте `deploy.config`** в корне проекта (скопируйте из `deploy.config.example`), укажите `SERVER_USER`, `SERVER_IP`, `SERVER_PORT`, `SERVER_PATH`.

2. **Установите зависимости** (если ещё не ставили): `npm install`

3. **Запустите настройку сервера** (пароль сервера — только из переменной окружения, в файл не писать):
   - **Windows (PowerShell):**  
     `$env:SSH_PASSWORD="ваш_пароль_сервера"; npm run setup:server`
   - **Windows (CMD):**  
     `set SSH_PASSWORD=ваш_пароль_сервера && npm run setup:server`
   - **Linux/macOS:**  
     `SSH_PASSWORD=ваш_пароль_сервера npm run setup:server`

   Скрипт по SSH установит PostgreSQL (если нет), создаст пользователя БД `streamdesk_user`, базу `streamdesk`, права, файл `.env` на сервере, при необходимости Node.js и PM2. В конце выведет сгенерированные `DATABASE_URL` и `SESSION_SECRET` — их можно сохранить у себя.

4. **Деплой приложения:**  
   `npm run deploy`

   Код соберётся, загрузится на сервер, выполнится `db:push`, сборка и перезапуск PM2.

После этого приложение будет доступно на сервере по порту 5000 (или по настройкам Nginx, если его поднимете).

---

## 1. Подключение по SSH (вручную)

Используйте ваш хост, порт и учётные данные (пароль не храните в репозитории — задавайте вручную или через переменную окружения `SSH_PASSWORD` / SSH-ключ).

Пример (подставьте свой пароль при запросе):

```bash
ssh deploy@203.0.113.10
```

На Windows в PowerShell или CMD пароль вводится при запросе. Рекомендуется настроить SSH-ключ, чтобы не вводить пароль при каждом деплое.

---

## 2. Установка на сервере (если ещё не установлено)

### Node.js (LTS)

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### PostgreSQL

```bash
sudo apt-get update
sudo apt-get install -y postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

---

## 3. База данных PostgreSQL

Создайте пользователя и базу (пароль замените на свой надёжный):

```bash
sudo -u postgres psql
```

В консоли PostgreSQL:

```sql
CREATE USER streamdesk_user WITH PASSWORD 'ваш_пароль';
CREATE DATABASE streamdesk OWNER streamdesk_user;
GRANT ALL PRIVILEGES ON DATABASE streamdesk TO streamdesk_user;
\q
```

Проверка: PostgreSQL по умолчанию слушает `localhost:5432` — для приложения на том же сервере этого достаточно.

---

## 4. Проект и переменные окружения

Загрузите проект в каталог (например `/opt/streamdesk`): через `git clone`, `npm run deploy` или копирование файлов.

Создайте на сервере файл `.env` в корне проекта:

```env
# База данных (приложение и БД на одном сервере — host localhost)
DATABASE_URL=postgresql://streamdesk_user:ваш_пароль@localhost:5432/streamdesk

PORT=5000
NODE_ENV=production

# Обязательно в production: длинная случайная строка (например: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")
SESSION_SECRET=ваша_длинная_случайная_строка_32_символа_и_больше

# Отключить резервный вход администратора без БД (в production всегда false)
ALLOW_FALLBACK_ADMIN=false

# YouGile (таск-менеджер): ключ из настроек YouGile
# Лимит API YouGile: 50 запросов/мин — в приложении уже есть очередь и повтор при 429
YOUGILE_API_KEY=ваш_ключ_yougile
```

Опционально для YouGile:

```env
YOUGILE_BASE_URL=https://ru.yougile.com/api-v2
YOUGILE_DEFAULT_COLUMN_ID=id_колонки_для_новых_задач
```

## 5. Сборка и миграции

На сервере в каталоге проекта:

```bash
cd /opt/streamdesk
npm install
npm run db:push
npm run build
```

`db:push` создаёт/обновляет таблицы в PostgreSQL.

---

## 6. Запуск (PM2)

```bash
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

Проверка: `pm2 status`, логи: `pm2 logs streamdesk`.

---

## 7. Регистрация пользователей

- Регистрация уже включена: пользователи отправляют заявку через форму, создаётся учётная запись с `active: false`.
- Администратор в панели подтверждает пользователя (активирует) — после этого пользователь может войти.
- Первый вход: задайте `ADMIN_USERNAME` и `ADMIN_PASSWORD`, выполните сидирование при необходимости либо создайте администратора вручную через БД/скрипт.

В production обязательно смените пароль админа и задайте `SESSION_SECRET`, `ALLOW_FALLBACK_ADMIN=false`.

---

## 8. YouGile: лимиты API и очередь

- Лимит YouGile: **50 запросов в минуту** на компанию.
- В приложении уже реализовано:
  - Ограничение исходящих запросов (до 48/мин) и кэш ответов.
  - При ответе 429 («слишком много запросов») полная синхронизация и мутации (создание/обновление/удаление задач) ставятся в очередь и повторяются через указанное API время (или через 1–2 минуты).
- Дополнительно настраивать ничего не нужно — достаточно указать `YOUGILE_API_KEY` в `.env`.

---

## 9. Деплой с вашей машины (с нестандартным портом SSH)

Скопируйте `deploy.config.example` в `deploy.config` и укажите свои данные (пароль в файл не пишите):

```env
SERVER_USER=deploy
SERVER_IP=203.0.113.10
SERVER_PORT=22
SERVER_PATH=/opt/streamdesk
```

Деплой:

```bash
npm run deploy
```

На Windows при отсутствии rsync используется SCP; для нестандартного SSH-порта укажите соответствующий `SERVER_PORT` в `deploy.config`.

---

## 10. Краткий чеклист

- [ ] PostgreSQL установлен, созданы пользователь и база `streamdesk`
- [ ] В `.env` на сервере: `DATABASE_URL`, `SESSION_SECRET`, `ALLOW_FALLBACK_ADMIN=false`
- [ ] Выполнены `npm run db:push` и `npm run build`
- [ ] Приложение запущено через PM2
- [ ] (Опционально) YouGile: `YOUGILE_API_KEY` в `.env` — синхронизация и очередь по лимитам работают автоматически

После этого все пользователи могут регистрироваться (с подтверждением админом), а синхронизация с YouGile учитывает лимиты API и работает через очередь.
