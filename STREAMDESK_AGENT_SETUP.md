# StreamDesk Agent setup

Агент нужен, чтобы ПК, сервер или vMix-машина сами появлялись в CRM и обновляли IP/метрики без ручной привязки.

## Файлы

- `agent-computer.bat` - обычный ПК.
- `agent-server.bat` - сервер.
- `agent-vmix.bat` - машина с vMix, мониторинг плюс выполнение расписания.
- `scripts/streamdesk-agent.ps1` - общий PowerShell-агент, который запускают `.bat` файлы.

Если переносите агент на другую машину, сохраните такую же структуру: `.bat` рядом с папкой `scripts`.

## CRM

В `.env` CRM можно задать:

```env
STREAMDESK_AGENT_TOKEN=long-random-token
```

В development без токена агент разрешен для быстрого локального теста. Для сервера, который доступен из сети, лучше задать токен и вставить тот же токен при запуске `.bat`.

## Запуск на ПК или сервере

1. Запустите `agent-computer.bat` или `agent-server.bat`.
2. Введите URL CRM, например `http://192.168.1.10:5000` или домен CRM.
3. Если на CRM задан `STREAMDESK_AGENT_TOKEN`, вставьте токен.
4. Машина появится в `Мониторинг системы`, а ПК также будет создан в разделе `Компьютеры`.

## Запуск рядом с vMix

1. В vMix включите Web Controller/API в `Settings -> Web`.
2. Запустите `agent-vmix.bat` на той же машине.
3. По умолчанию агент читает vMix тут: `http://127.0.0.1:8088/api`.
4. В расписателе vMix можно оставить host пустым или `localhost`, тогда событие заберет vMix-агент. Если vMix-агентов несколько, укажите в поле host agentKey нужной машины.

## Dropped frames

Агент отправляет счетчики dropped frames, если они доступны в ответе vMix API. В стандартном XML vMix они могут отсутствовать, поэтому в CRM тогда будет `-`. Для гарантированного показа Source Dropped / Renderer Dropped / Resync можно добавить отдельный сбор из логов или vMix script API.
