# Architectural Decision Records (ADRs)

This section contains the Architectural Decision Records (ADRs) for the StreamDesk project.

ADRs document important architectural and technical decisions made during the project's development. Each record describes the context in which a decision was made, the chosen solution, and its expected consequences.

## Available ADRs

- **ADR-001 – Centralized Equipment Permissions**
  
  Defines the centralized permission model for equipment management and access control.

- **ADR-002 – Declarative Protected Route Wrapper**
  
  Describes the approach for protecting application routes using a declarative wrapper.

- **ADR-003 – Unified Monorepo Coverage**
  
  Explains the unified strategy for collecting and reporting code coverage across the monorepo.

- **ADR-004 – Authenticated Scoped Realtime Transport**

  Defines session-authenticated WebSocket upgrades, authorized scope subscriptions, identifier-only events, and reconnect/refetch recovery.
