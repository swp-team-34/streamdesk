# Week 3 Sprint Retrospective

## Scope and format

- Sprint or period covered: Week 3.
- Sprint goal or focus: Prepare the backlog and workflow for MVP v1.
- Participants represented in this retrospective: StreamDesk assignment team.

## What went well

1. We now have clear workflow rules for issues, branches, PRs, releases, and evidence.
2. The main issue templates are ready, which makes backlog work easier to create consistently.
3. Week 2 already gave us a prioritized set of stories, so we did not need to start from zero.

## What did not go well

1. We spent extra time deciding which features should really stay in the first MVP.
2. Some backlog items still need clearer boundaries before they are easy to estimate.
3. A few requirement details still need customer clarification before development can move faster.

## Action points

1. Agree on a smaller and clearer MVP v1 scope before the next planning session.
2. Split the larger backlog items and clarify open requirement questions before estimation.
